# Python 路线对照表

> 用来快速理解 30 天学习路线中每个阶段解决什么问题、适合什么场景。

## 阶段对照

| Phase | 天数 | 主题 | 解决的问题 |
| --- | --- | --- | --- |
| Phase 1 | Day 1 - 7 | 核心语法 | 建立 Python 语法、数据结构、函数基础 |
| Phase 2 | Day 8 - 12 | OOP 与模块化 | 学会组织更真实的项目代码 |
| Phase 3 | Day 13 - 18 | 异步与 API | 学会接入服务、编写接口、处理实时通信 |
| Phase 4 | Day 19 - 23 | 数据与自动化 | 学会文本处理、数据分析、可视化和办公自动化 |
| Phase 5 | Day 24 - 28 | AI Agent 核心 | 学会调用模型、写 Prompt、接工具、做 RAG |
| Phase 6 | Day 29 - 30 | 工程化与部署 | 学会测试、日志、配置和 Docker 部署 |

## 推荐路径

### 如果你主要目标是 AI 应用

按 `Phase 1 -> Phase 2 -> Phase 3 -> Phase 5 -> Phase 6` 学。

### 如果你主要目标是自动化和数据处理

按 `Phase 1 -> Phase 2 -> Phase 4 -> Phase 6` 学。

### 如果你主要目标是后端接口开发

按 `Phase 1 -> Phase 2 -> Phase 3 -> Phase 6` 学。

## 与 Rust 路线的差异

- Python 路线更偏快速落地，强调工具使用、库生态和 AI 应用。
- Rust 路线更偏语言原理、类型系统、所有权和性能模型。
- 如果你想先做出可运行产品，优先 Python；如果想补底层工程能力，再补 Rust。
